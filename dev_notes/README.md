# dev_notes/ — 앱 Dev Mode 캡처 (에이전트가 읽는 작업 큐)

이 폴더의 `*.md` 파일은 **앱 개발자 모드**에서 사용자가 "여기 고쳐줘"라고 캡처한 항목이다.
각 파일 하나가 **고칠 부분의 위치 + 소스(파일:라인) + 요청(메모)** 를 한 번에 담는다.

## 에이전트 사용법 (처리 프로토콜)
1. 작업 시작 시 `dev_notes/` 의 `*.md`(README·`done/` 제외)를 확인 — **미처리 큐**.
2. 각 노트의 **🧩 소스 후보(file:line)** 를 먼저 부분 Read. 없으면 **🔎 Grep 앵커** 로 찾는다.
   (마크업=`scripts/ui_templates.py`, 동작/라우트/주입=`scripts/market_dashboard3_realtime.py`.)
3. 세션 파일은 **TODO 전체를 한 계획으로 일괄** 수정 → `uv run scripts/smoke_check.py`(+수집기면 `api_smoke.py`) 통과.
4. **처리 내용을 그 파일에 반영**한다:
   - 끝낸 항목 체크박스 `[ ]`→`[x]` (못 한 건 `[ ]` + 사유).
   - 맨 아래 **`## ✅ 처리 결과`** 채우기: 바꾼 file:line · `changes_X_*.md` 로그 id · smoke 결과.
5. **전부 끝났으면** 파일을 `dev_notes/done/` 로 **이동**(완료 큐 분리 → 다음 세션은 미처리만 봄).
   일부만 끝냈으면 `dev_notes/` 에 그대로 두고 결과만 갱신. (`done/` 은 첫 완료 시 `mkdir`; 사용자가 재확인 후 삭제 가능.)

## 파일 종류
- `session_<날짜>_<이름>.md` — **세션(권장)**: 여러 수정 요청을 묶은 TODO 체크리스트. 한 번에 처리용.
- `<시각>_<슬러그>.md` — 단건 노트(`💾 즉시 저장`).

## 만드는 법 (사용자)
- 앱을 `KMKT_DEV=1 uv run application_build/app.py` 로 실행(또는 `KMKT_DEV=1` 환경).
- 앱에서 **⌘⇧D**(Ctrl+Shift+D) → 개발자 모드 ON(시안색 틀 + DEV 배지).
- 고칠 요소를 **클릭(또는 우클릭)** → 팝오버에 위치·소스·툴 표시 → **메모** 입력.
  - **여러 개를 한 번에**: `➕ 세션 추가` 로 모으고(배지 `📌 N` 카운터), 배지 `📌` 클릭 → 패널에서
    세션 이름 정하고 **💾 세션 저장** → `session_*.md` 하나로 저장. → 에이전트에게 "이 세션 처리해줘".
  - **한 건만**: `💾 즉시 저장`.
- 배지의 `🖱 검사` 를 끄면 앱을 평소처럼 조작(이동·네비게이션) 가능, 다시 켜서 캡처.

> 목적: 고칠 부분의 위치·기능·내용을 **한 파일로 통합 전달** → 에이전트가 헤매지 않고(토큰 절약) 바로 작업.
> 동작 정의: `scripts/dev_overlay.py`, 라우트 `/api/dev/locate`·`/api/dev/note` (KMKT_DEV=1 일 때만).
